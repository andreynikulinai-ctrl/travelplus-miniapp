# Где искать ошибку при отправке заявки

## 1. Терминал бэкенда (где запущен `node server.js`)

После отправки формы смотри этот терминал:

- **Появилось** `[backend] POST /api/send получен` → запрос доходит до бэкенда, проблема дальше (Telegram или ответ).
- **Ничего не появилось** → запрос не доходит до бэкенда (прокси или бэкенд не на 3001).

Дальше в логах может быть:
- `[backend] Заявка отправлена в Telegram` → всё ок.
- `[backend] Ошибка Telegram: {...}` → ошибка от Telegram (токен, chat_id, сеть).
- `[backend] sendToTelegram error: ...` → падение в коде бэкенда.

## 2. Консоль браузера (DevTools)

1. Открой сайт http://localhost:5173
2. Правый клик → «Просмотреть код» / «Inspect» → вкладка **Console**
3. Отправь форму ещё раз
4. Найди строку `[форма] Ответ сервера: ...` — там код ответа (200, 404, 500) и начало тела ответа

Что значит:
- **404** — бэкенд не получил запрос (прокси или не тот URL).
- **500** — бэкенд ответил ошибкой; в той же строке будет текст ошибки.
- **200** и `{"success":true}` — успех; если всё равно показывается ошибка, значит баг в обработке ответа во фронте.

## 3. Чек-лист

- [ ] Бэкенд запущен: в одном терминале `cd ../backend && node server.js`, в конце есть «Backend ready on http://localhost:3001».
- [ ] Фронтенд запущен: в другом терминале `npm run dev` из папки `frontend`, открыт http://localhost:5173.
- [ ] Отправляешь форму с того же сайта (localhost:5173), а не с файла file://.

После проверки пришли:
1. Что именно появилось (или не появилось) в терминале бэкенда.
2. Текст строки `[форма] Ответ сервера: ...` из консоли браузера.

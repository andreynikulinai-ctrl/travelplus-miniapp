# Bitrix24 — интеграция заявок (к вернуться позже)

**Контекст (обсуждение с юристами):**  
Первичная обработка ПДн сейчас происходит на стороне Vercel (не в РФ). Юридически чувствительно: первая точка приёма заявки — за рубежом. Решение: получать заявки **изначально в Битрикс24**, а не в Telegram-бота.

---

## Что запланировано сделать позже

1. **Настроить отправку заявок в Bitrix24** вместо (или в дополнение к) Telegram:
   - endpoint `/api/send` перевести на создание лида в Bitrix24 (REST: `crm.lead.add`);
   - в лид: имя, телефон, email, компания, комментарий, список выбранных товаров.

2. **Решить по Telegram:**
   - полностью убрать отправку ПДн в Telegram, **или**
   - оставить только короткое уведомление без персональных данных (например: «Новый лид в Битрикс24, ID …»).

3. **Варианты архитектуры (уже обсуждали):**
   - **Вариант A:** оставить Vercel, но логику `/api/send` — только Bitrix24 (или Bitrix24 + опционально уведомление в TG). Первая техническая точка по-прежнему Vercel; первый «получатель» ПДн как бизнес-сущности — Битрикс24.
   - **Вариант B:** первая точка приёма в РФ — форма шлёт на свой сервер/хостинг в РФ или напрямую в вебхук Bitrix24 (инфраструктура + при необходимости CORS).

4. **Что понадобится для настройки:**
   - URL вебхука Bitrix24 (например: `https://ПОРТАЛ.bitrix24.ru/rest/ID/WEBHOOK/crm.lead.add.json`);
   - список полей лида, которые нужно заполнять;
   - финальное решение: оставляем ли уведомления в Telegram и в каком виде.
